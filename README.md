<h1>Автоматизация сбора данных через Telegram Mini App</h1>

<h3>О проекте</h3>
Проект автоматизации процесса сбора и обработки данных через Telegram Mini App с последующей передачей информации в Google Sheets.  
<b>Цель проекта</b> — исключить ручной ввод данных и создать единый поток информации для дальнейшей аналитики.

#

<h3>Бизнес-задача</h3>

<b>До внедрения:</b>

- данные передавались вручную
- были ошибки ввода
- отсутствовал единый формат
- сложность контроля

<b>После внедрения:</b>

- единая форма
- автоматическая запись
- контроль статусов
- прозрачность процесса

#

<h3>Используемые инструменты</h3>

- Telegram Mini App
- Google Forms
- Google Sheets
- Google Apps Script
- Process automation
- Business analysis

#

<h3>Схема процесса</h3>

<b>Анализ процесса взаимодействия систем</b>

Ниже представлена схема логики взаимодействия пользователя, Telegram Mini App и Google Sheets в рамках автоматизации сбора данных.

<b>Логика процесса</b>

1. Пользователь открывает Telegram Mini App.
2. Через встроенную форму вводит необходимые данные.
3. Система выполняет валидацию введённой информации.
4. При наличии ошибок пользователь получает уведомление и может скорректировать данные.
5. При успешной проверке информация автоматически передается в Google Sheets.
6. После записи данных автоматически обновляется управленческий dashboard.
7. Руководитель получает актуальные показатели без ручной обработки.

<b>Задачи системного анализа в проекте</b>

В рамках проекта были определены:

- точки входа данных
- маршрут передачи информации
- правила валидации
- структура хранения данных
- сценарии обработки ошибок
- логика обновления отчётности

<b>Бизнес-эффект<b>

Реализация процесса позволила:

- сократить ручной ввод данных
- снизить количество ошибок
- ускорить обновление отчётности
- повысить прозрачность процессов
- уменьшить операционную нагрузку

<h3>Моя роль в проекте</h3>

В рамках проекта выполнял функции системного анализа и проектирования решения:

- анализировал текущий процесс сбора данных
- выявлял узкие места ручной обработки
- формировал требования к пользовательской форме
- определял структуру передаваемых данных
- проектировал маршрут движения информации
- описывал сценарии обработки ошибок
- участвовал в проектировании логики обновления dashboard
- контролировал корректность итогового решения

Основная задача заключалась в том, чтобы перевести ручной процесс в автоматизированный и прозрачный workflow.

<h3>Результат проекта</h3>

После внедрения решения удалось:

- сократить время передачи данных
- исключить двойной ввод информации
- уменьшить количество ошибок
- ускорить обновление KPI
- повысить прозрачность для руководителя
